Schedule Assistant Sync Endpoints UPDATED v1.7

Three endpoints receive automated data from Schedule Assistant (SA, port 4400). All /api/sync paths bypass auth middleware for internal SA-to-TA calls. SA now runs a 6-rule SLA engine (R6 added in v1.7) with 12-point security hardening.

MethodPathSourceDB WritePurpose
POST/api/sync/scheduleSA auto-chainScheduleEntry (upsert)Receive schedule entries from SA SLA engine. Match by woNumber + scheduledDate.
POST/api/sync/work-ordersSA auto-chainWorkOrder (upsert)Receive work order data from SA. Match by woNumber (unique). Create or update.
POST/api/sync/write-to-saasSA auto-chainWorkOrder (update status)Write WO status back to mock Salesforce. Closes the automation loop: queued → scheduled.

Auto-Sync Pipeline (Full Loop)

SaaS (Salesforce)
    |
    v  POST /api/sync/pull-queue (SA)
Schedule Assistant (SA, port 4400)
    |  6-rule SLA engine (R6 → R5 → R1 → R2 → R3 → R4)
    |  R6: priority=3 same-day emergency dispatch
    v  POST /api/sync/schedule + /api/sync/work-orders
Technician Assistant (TA, port 4300)
    |  Upserts schedule + WO records
    |  R6 entries appear as red-pulse chips on calendar
    v  POST /api/sync/write-to-saas
SaaS (Salesforce)
    WO status: queued -> scheduled

The entire pipeline runs automatically when SA pulls new work orders from SaaS. No manual intervention required. Techs see their assignments on TA's Schedule page.

R6 — High Priority Emergency Dispatch (v1.7)

New SLA rule runs first in evaluation order (before R5). Triggers on priority=3 work orders.

ConditionResult
Queued before noon local time AND within 50 mi of a techSame-day dispatch (uses EVE slot, overrides 2-job limit)
Queued after noon OR weekendNext business day, first available slot

Calendar chips for R6 entries are red with pulse animation. Haversine distance calculation (same as R1/R5) determines closest eligible tech.
